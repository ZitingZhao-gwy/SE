# FSE 股票交易系统总用户手册

本手册面向整套系统的演示者、联调人员和使用人员，说明每个子系统大致负责什么、通常从哪里进入、登录后能做什么。

本手册讲“怎么用系统”，不讲源码实现细节。

## 1. 系统组成

整套系统目前包含五个主要子系统：

1. 账户管理子系统 `account-management`
2. 中央交易子系统 `central-trading`
3. 网上信息发布子系统 `online-info-publish`
4. 交易管理子系统 `trade-management`
5. 交易客户端子系统 `trading-client`

其中可以按“工作人员侧”和“投资者侧”理解。

工作人员侧主要包括：

- 账户管理
- 交易管理

投资者侧主要包括：

- 交易客户端
- 网上信息发布

中央交易子系统主要负责后台撮合与交易处理。

## 2. 各子系统作用

### 2.1 账户管理子系统

主要由工作人员使用。

可完成：

- 证券账户开户、挂失、补办、销户
- 资金账户开户、存款、取款、改密、挂失、补办、销户
- 投资者信息修改
- 操作日志查询
- 对外提供资金快照、持仓快照、交易落账、冻结审计等接口

### 2.2 交易客户端子系统

主要由投资者使用。

可完成：

- 投资者登录
- 查看账户与交易相关信息
- 进行交易类操作
- 查询提醒与通知

### 2.3 交易管理子系统

主要由管理员或审核人员使用。

可完成：

- 审核相关操作
- 黑名单管理
- 交易管理侧业务处理

### 2.4 网上信息发布子系统

主要面向外部展示。

可完成：

- 公共信息展示
- 市场信息查看
- 从发布页面跳转到其他投资者使用页面

### 2.5 中央交易子系统

主要是后台核心处理模块。

可完成：

- 委托处理
- 撮合成交
- 成交结果广播
- 行情与订单事件处理

## 3. 常见使用路径

### 3.1 工作人员办理账户业务

典型流程如下：

1. 进入账户管理子系统登录页
2. 输入工作人员账号密码
3. 如果是首次登录，完成证书认证
4. 进入 Dashboard
5. 从 Dashboard 或账户列表页面进入具体业务
6. 办理证券账户或资金账户业务
7. 在 Dashboard 查看操作日志

### 3.2 投资者查看信息并交易

典型流程如下：

1. 先从网上信息发布页面查看公开信息
2. 需要交易时，进入交易客户端
3. 登录投资者账户
4. 查看账户与交易数据
5. 发起委托、撤单、查询等操作

### 3.3 管理侧处理审查或冻结问题

典型流程如下：

1. 进入交易管理或管理员相关页面
2. 登录管理人员账号
3. 查询目标对象
4. 进行审核、冻结、解冻或审查相关操作

## 4. 账户管理系统的使用关系

账户管理是整套系统里和账户生命周期最直接相关的模块。

它与其他模块的关系可以这样理解：

- 投资者是否有证券账户、资金账户，由账户管理决定
- 交易客户端做交易前，要依赖账户管理提供的资金与持仓信息
- 中央交易系统成交后，要把资金变化和持仓变化回写到账户管理
- 管理员冻结、审计、年结息等操作，也依赖账户管理的数据

## 5. 工作人员侧使用建议

对工作人员而言，最常用的是账户管理系统。

建议按下面顺序使用：

1. 先在证券账户模块完成证券开户
2. 再在资金账户模块完成资金开户
3. 默认建立证券账户与资金账户绑定关系
4. 需要存取款、改密、挂失、补办、销户时，再进入对应页面
5. 通过 Dashboard 查看统计信息和操作日志

## 6. 投资者侧使用建议

对投资者而言，通常不会直接操作账户管理系统。

更自然的路径是：

1. 在网上信息发布系统查看公开信息
2. 在交易客户端登录与操作
3. 由交易客户端和后台系统去调用账户管理、中央交易等接口
