# 对外部子系统接口说明

## 1. 说明

本文只整理**当前版本真正面向外部子系统**的接口。

这里的“外部子系统”包括：

- 交易客户端
- 中央交易系统

这里**不包括**你们自己内部的证券账户模块和资金账户模块之间的调用，因此：

- `bindSecurityAccount`
- `unbindSecurityAccount`

不计入对外部子系统接口。

同时，结合当前版本边界，以下内容也不纳入本文：

- 黑名单相关接口
- 管理员冻结/解冻接口
- 管理员审计接口
- 角色权限相关接口

## 2. 当前版本需要保留的外部接口

当前版本建议保留的对外部子系统接口共有 6 个：

1. `clientLoginAuth`
2. `getFundSnapshot`
3. `getSecuritySnapshot`
4. `clientChangeFundPassword`
5. `updateFundBalance`
6. `updateSecurityHolding`

其中：

- 前 4 个主要提供给**交易客户端**
- 后 2 个主要提供给**中央交易系统**

## 3. 交易客户端接口

## 3.1 `clientLoginAuth`

### 调用方

- 交易客户端

### 用途

- 投资者通过资金账户号和交易密码登录交易客户端

### 输入参数

| 参数名 | 类型 | 说明 |
|---|---|---|
| `fund_acc_no` | `VARCHAR(20)` | 资金账户号 |
| `trade_password` | `VARCHAR(128)` 或明文经后端统一处理后值 | 交易密码 |

### 返回结果

| 字段名 | 说明 |
|---|---|
| `code` | 结果码，`0` 表示成功 |
| `message` | 结果说明 |
| `fund_acc_no` | 资金账户号 |
| `sec_acc_no` | 绑定证券账户号 |
| `status` | 资金账户状态 |

### 数据库涉及

- `fund_account`

### 主要读取字段

- `fund_account.fund_acc_no`
- `fund_account.trade_password`
- `fund_account.sec_acc_no`
- `fund_account.status`

### 规则说明

- 先校验资金账户是否存在
- 再校验账户状态是否允许登录
- 再校验交易密码

## 3.2 `getFundSnapshot`

### 调用方

- 交易客户端

### 用途

- 查询资金账户快照，用于余额展示或下单前查看资金情况

### 输入参数

| 参数名 | 类型 | 说明 |
|---|---|---|
| `fund_acc_no` | `VARCHAR(20)` | 资金账户号 |

### 返回结果

| 字段名 | 说明 |
|---|---|
| `code` | 结果码 |
| `message` | 结果说明 |
| `available_balance` | 可用余额 |
| `frozen_balance` | 冻结余额 |
| `currency` | 币种 |
| `status` | 账户状态 |
| `recent_logs` | 最近资金流水摘要 |

### 数据库涉及

- `fund_account`
- `fund_transaction_log`

### 主要读取字段

- `fund_account.available_balance`
- `fund_account.frozen_balance`
- `fund_account.currency`
- `fund_account.status`
- `fund_transaction_log.txn_type`
- `fund_transaction_log.amount`
- `fund_transaction_log.txn_time`

### 规则说明

- 先看 `fund_account.status`
- 只要账户存在，就可以返回账户快照
- 是否允许继续交易，由调用方结合 `status` 再判断

## 3.3 `getSecuritySnapshot`

### 调用方

- 交易客户端

### 用途

- 查询证券账户持仓信息
- 卖出前校验某只股票的可卖数量

### 输入参数

| 参数名 | 类型 | 说明 |
|---|---|---|
| `sec_acc_no` | `VARCHAR(20)` | 证券账户号 |
| `stock_code` | `VARCHAR(10)`，可选 | 股票代码 |

### 返回结果

| 字段名 | 说明 |
|---|---|
| `code` | 结果码 |
| `message` | 结果说明 |
| `holdings` | 持仓列表或单只持仓 |
| `quantity` | 持有股数 |
| `frozen_quantity` | 冻结股数 |
| `available_quantity` | 可卖数量 |
| `avg_cost` | 每股成本 |

### 数据库涉及

- `security_account`
- `holding`

### 主要读取字段

- `security_account.sec_acc_no`
- `security_account.status`
- `holding.stock_code`
- `holding.quantity`
- `holding.frozen_quantity`
- `holding.avg_cost`

### 规则说明

- 先检查证券账户是否存在
- 再根据需要返回全部持仓或单只持仓
- 可卖数量为计算值：

```text
available_quantity = quantity - frozen_quantity
```

## 3.4 `clientChangeFundPassword`

### 调用方

- 交易客户端

### 用途

- 投资者修改交易密码或取款密码

### 输入参数

| 参数名 | 类型 | 说明 |
|---|---|---|
| `fund_acc_no` | `VARCHAR(20)` | 资金账户号 |
| `password_type` | `VARCHAR(20)` | `trade` 或 `withdraw` |
| `old_password` | `VARCHAR(128)` 或统一处理后值 | 原密码 |
| `new_password` | `VARCHAR(128)` 或统一处理后值 | 新密码 |

### 返回结果

| 字段名 | 说明 |
|---|---|
| `code` | 结果码 |
| `message` | 结果说明 |

### 数据库涉及

- `fund_account`
- `operation_log`

### 主要读取字段

- `fund_account.trade_password`
- `fund_account.withdraw_password`
- `fund_account.status`

### 主要更新字段

- `fund_account.trade_password`
或
- `fund_account.withdraw_password`

### 规则说明

- 先检查账户状态
- 再校验原密码
- 最后更新对应密码字段
- 关键密码修改应记录到 `operation_log`

## 4. 中央交易系统接口

## 4.1 `updateFundBalance`

### 调用方

- 中央交易系统

### 用途

- 买入下单时冻结资金
- 买入成交时扣减冻结资金
- 卖出成交时增加可用余额
- 撤单时释放冻结资金

### 输入参数

| 参数名 | 类型 | 说明 |
|---|---|---|
| `fund_acc_no` | `VARCHAR(20)` | 资金账户号 |
| `ref_order_id` | `VARCHAR(50)` | 关联订单号 |
| `txn_type` | `VARCHAR(20)` | `买入冻结` / `买入扣款` / `卖出回款` / `撤单解冻` |
| `amount` | `DECIMAL(15,2)` | 变动金额 |

### 返回结果

| 字段名 | 说明 |
|---|---|
| `code` | 结果码 |
| `message` | 结果说明 |
| `available_balance` | 更新后可用余额 |
| `frozen_balance` | 更新后冻结余额 |
| `log_id` | 资金流水编号 |

### 数据库涉及

- `fund_account`
- `fund_transaction_log`

### 主要读取字段

- `fund_account.status`
- `fund_account.available_balance`
- `fund_account.frozen_balance`
- `fund_transaction_log.ref_order_id`

### 主要更新字段

- `fund_account.available_balance`
- `fund_account.frozen_balance`
- `fund_transaction_log.txn_type`
- `fund_transaction_log.amount`
- `fund_transaction_log.available_after`
- `fund_transaction_log.frozen_after`
- `fund_transaction_log.ref_order_id`

### 规则说明

- 这是当前版本最重要的交易协同接口之一
- 账户状态必须先检查
- 然后再检查余额或冻结余额是否足够
- 必须基于 `ref_order_id` 做幂等控制
- 成功后必须写入资金流水

### 对应数据库事务动作

- `买入冻结`：`available_balance` 减少，`frozen_balance` 增加
- `买入扣款`：`frozen_balance` 减少
- `卖出回款`：`available_balance` 增加
- `撤单解冻`：`frozen_balance` 减少，`available_balance` 增加

## 4.2 `updateSecurityHolding`

### 调用方

- 中央交易系统

### 用途

- 买入成交后增加持仓
- 卖出下单时冻结持仓
- 卖出成交时扣减持仓和冻结股数
- 撤单时释放冻结股数

### 输入参数

| 参数名 | 类型 | 说明 |
|---|---|---|
| `sec_acc_no` | `VARCHAR(20)` | 证券账户号 |
| `stock_code` | `VARCHAR(10)` | 股票代码 |
| `change_type` | `VARCHAR(20)` | `买入增加` / `卖出冻结` / `卖出扣减` / `撤单释放` |
| `quantity` | `INT` | 变动数量 |
| `price` | `DECIMAL(15,4)`，可选 | 买入时用于更新成本 |

### 返回结果

| 字段名 | 说明 |
|---|---|
| `code` | 结果码 |
| `message` | 结果说明 |
| `quantity` | 更新后持有股数 |
| `frozen_quantity` | 更新后冻结股数 |
| `avg_cost` | 更新后平均成本 |

### 数据库涉及

- `security_account`
- `holding`

### 主要读取字段

- `security_account.status`
- `holding.quantity`
- `holding.frozen_quantity`
- `holding.avg_cost`

### 主要更新字段

- `holding.quantity`
- `holding.frozen_quantity`
- `holding.avg_cost`
- `holding.updated_at`

### 规则说明

- 这是当前版本最重要的交易协同接口之一
- 必须先检查证券账户状态
- 卖出冻结时检查可卖数量是否充足
- 卖出扣减时检查冻结股数和持有股数是否足够
- 买入增加时若原持仓不存在，可以新增持仓记录
- 买入增加时需要更新移动平均成本

## 5. 与设计报告中的协同关系对应

根据设计报告内容，当前版本真正保留的系统协同关系主要是：

- 交易客户端 -> 资金账户业务
  - `clientLoginAuth`
  - `getFundSnapshot`
  - `clientChangeFundPassword`
- 交易客户端 -> 证券账户业务
  - `getSecuritySnapshot`
- 中央交易系统 -> 资金账户业务
  - `updateFundBalance`
- 中央交易系统 -> 证券账户业务
  - `updateSecurityHolding`

这几项正对应了设计报告中提到的两个重点协同场景：

1. 投资者在交易客户端登录、查余额、查持仓、改密码
2. 中央交易系统在撮合后更新资金和持仓
